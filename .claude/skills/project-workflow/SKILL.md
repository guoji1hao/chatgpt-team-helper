# Project Workflow

在本项目中工作时，默认遵守以下规则：

1. 使用 Git 作为版本管理工具。
2. 本地开发时，每次任务完成后都要提交一次 Git commit。
3. 可以自主使用网络搜索、Claude Code 工具，以及安装推进任务所需的软件、依赖或辅助工具。
4. 遇到问题先自行排查和解决；只有涉及不可逆操作、共享外部系统、人工登录或关键业务决策时再询问用户。
5. 每次任务完成后，更新同目录下的 `LESSONS.md`，记录本次任务的经验和踩坑。

## 工作方式

- 先理解现状，再修改代码或环境。
- 优先修改已有文件，不为未来假设做额外抽象。
- 只暂存与当前任务相关的文件。
- 先本地验证，再提交 Git。
- 提交信息要简洁，突出改动目的。

## 经验沉淀格式

每次任务完成后，在 `LESSONS.md` 中追加一段：

```md
## YYYY-MM-DD — 任务标题
- 任务：
- 有效做法：
- 踩坑：
- 复用规则：
```

## 使用时机

当需要在这个项目里持续开发、修复问题、推进任务，或者需要回顾本项目既定工作方式时，使用这个 skill。
